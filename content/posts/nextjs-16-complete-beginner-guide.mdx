### Plain React vs Next.js — Difference Kya Hai?

| Feature | Plain React (Vite) | Next.js |
|---------|---------------------|---------|
| **Rendering** | Client-side only | Server + Client + Static |
| **SEO** | ❌ Hard (CSR) | ✅ Excellent (SSR/SSG) |
| **Routing** | Manual (react-router) | ✅ File-based |
| **API Routes** | ❌ Separate backend | ✅ Built-in |
| **Image Optimization** | Manual | ✅ Automatic (next/image) |
| **Production Build** | Basic | ✅ Optimized + caching |

<Callout type="success" title="Bottom Line">
**Plain React** = Simple SPAs ke liye.

**Next.js** = Real-world production apps (blogs, e-commerce, SaaS, dashboards) ke liye.
</Callout>

Next.js 14/15 se aage, Next.js 16 ne ye improvements diye:

### 🚀 Turbopack — Default Bundler

Webpack ki jagah **Turbopack** (Rust-based) default hai:

<Stat value="10x faster" label="Dev server startup" color="orange" />

<Stat value="2-3s" label="Hot reload typical" color="green" />

### ⚡ React 19 Support

React 19 ki **naye features** like:
- `useActionState()` — server actions ke saath form handling
- `use()` hook — promises directly mein read karna
- `useFormStatus()` — form submission state
- Improved server components

### 🎯 Better Caching Defaults

Next.js 16 mein **fetch caching default OFF hai**:

```javascript
// Next.js 14: cached by default (confusing)
fetch('https://api.example.com')

// Next.js 16: not cached by default (predictable)
fetch('https://api.example.com')

// Cache karna ho to explicit:
fetch('https://api.example.com', { cache: 'force-cache' })
```

### 🔧 Async Params (Breaking Change from Next.js 14)

Dynamic routes mein params ab **async** hain:

```javascript
// ❌ Next.js 14 (sync)
export default function Page({ params }) {
  return <div>{params.slug}</div>
}

// ✅ Next.js 16 (async)
export default async function Page({ params }) {
  const { slug } = await params;
  return <div>{slug}</div>
}
```

## 5. App Router — File-Based Routing

App Router mein **folder structure = URL structure** hai.

### Example:

```
app/
├── page.tsx              → /
├── about/
│   └── page.tsx          → /about
├── blog/
│   ├── page.tsx          → /blog
│   └── [slug]/
│       └── page.tsx      → /blog/anything
└── dashboard/
    └── settings/
        └── page.tsx      → /dashboard/settings
```

### Dynamic Routes:

`[slug]` mein kuch bhi aa sakta hai:

```javascript
// app/blog/[slug]/page.tsx

export default async function BlogPost({ params }) {
  const { slug } = await params;
  
  return (
    <article>
      <h1>Post: {slug}</h1>
    </article>
  );
}
```

URLs jo match honge:
- `/blog/my-first-post` → `slug = "my-first-post"`
- `/blog/react-tutorial` → `slug = "react-tutorial"`

### Special Files:

| File | Purpose |
|------|---------|
| **page.tsx** | Page UI (URL pe render) |
| **layout.tsx** | Persistent UI (har child page mein) |
| **loading.tsx** | Loading state (Suspense) |
| **error.tsx** | Error boundary |
| **not-found.tsx** | 404 page |
| **route.ts** | API endpoint |

## 6. Server vs Client Components — CRITICAL Concept

Ye Next.js 16 ka **most important concept** hai.

### Server Components (Default)

By default, **sab components Server Components hain**:

```jsx
// Server Component (default)
export default async function HomePage() {
  // Server pe run hota hai
  // Database call, file system, secrets safe hain
  
  const data = await fetch('https://api.example.com');
  
  return <div>{data.title}</div>;
}
```

**Server Components benefits**:
- ✅ Database directly call kar sakte ho
- ✅ Secrets/API keys safe hain (browser mein nahi jate)
- ✅ Faster initial load (HTML server pe ready)
- ✅ Smaller JavaScript bundle

**Limitations**:
- ❌ `useState`, `useEffect` nahi chal sakte
- ❌ Browser APIs (window, localStorage) nahi
- ❌ Event handlers (onClick) directly nahi

### Client Components (Opt-in)

Interactive UI ke liye `"use client"` directive use karo:

```jsx
"use client";  // ← Ye line CRITICAL hai

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

**Client Components needed when**:
- State use karna ho (`useState`)
- Effects (`useEffect`)
- Event handlers (`onClick`, `onChange`)
- Browser APIs (`window`, `localStorage`)
- Third-party libraries with hooks

<ProTip title="🎯 Best Practice">
**Server-first approach** lo. Pura page Server Component banao. Sirf interactive bits (button, form input, modal) ko Client Component banao. Performance massively improve hoti hai.
</ProTip>

## 7. Data Fetching Examples

### Server Component Mein Fetch (Most Common):

```jsx
// app/posts/page.tsx

async function getPosts() {
  const res = await fetch('https://api.example.com/posts');
  return res.json();
}

export default async function PostsPage() {
  const posts = await getPosts();
  
  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

### Database Direct Query:

```jsx
// app/users/page.tsx
import { db } from '@/lib/db';

export default async function UsersPage() {
  // Direct database query — secure!
  const users = await db.user.findMany();
  
  return (
    <div>
      {users.map(user => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}
```

### Caching Control:

```jsx
// Always fresh data (no cache)
fetch(url, { cache: 'no-store' });

// Cache forever (until rebuild)
fetch(url, { cache: 'force-cache' });

// Revalidate every 60 seconds
fetch(url, { next: { revalidate: 60 } });

// Default (Next.js 16): no cache
fetch(url);
```

## 8. React 19 Killer Features

### `useActionState()` — Form Handling Made Easy

```jsx
"use client";

import { useActionState } from 'react';
import { submitForm } from './actions';

export default function ContactForm() {
  const [state, formAction, isPending] = useActionState(
    submitForm,
    { message: '' }
  );
  
  return (
    <form action={formAction}>
      <input name="email" type="email" required />
      <button disabled={isPending}>
        {isPending ? 'Sending...' : 'Submit'}
      </button>
      {state.message && <p>{state.message}</p>}
    </form>
  );
}
```

### Server Actions:

```javascript
// app/actions.ts
"use server";

export async function submitForm(prevState, formData) {
  const email = formData.get('email');
  
  // Server pe directly run hota hai
  await sendEmail(email);
  
  return { message: 'Email sent successfully!' };
}
```

Ye **revolutionary** hai — koi API route banane ki zaroorat nahi.

## 9. Metadata API — SEO Built-In

Har page ke liye **metadata** export karo:

```jsx
// app/blog/[slug]/page.tsx

export async function generateMetadata({ params }) {
  const { slug } = await params;
  const post = await getPost(slug);
  
  return {
    title: post.title,
    description: post.description,
    openGraph: {
      title: post.title,
      images: [post.image],
    },
    twitter: {
      card: 'summary_large_image',
      title: post.title,
    },
  };
}

export default async function PostPage({ params }) {
  // page content
}
```

**SEO automatic**:
- `<title>` tag set ho jata hai
- Meta description
- Open Graph (Facebook/LinkedIn shares)
- Twitter cards
- Canonical URLs

## ⚠️ Common Mistakes — Avoid Karein

<Callout type="danger" title="❌ Top Mistakes Beginners Karte Hain">
1. **`use client` directive bhulna** — useState use karne pe error
2. **Server Component mein useState use karna** — error: hooks can only be used in Client Components
3. **Async params await nahi karna** (Next.js 16 mein)
4. **fetch caching assume karna** — explicitly specify karo
5. **Browser APIs Server Component mein use karna** — `window is not defined`
6. **API keys Client Component mein expose karna** — major security risk
7. **`<img>` tag use karna** — `next/image` use karo (auto-optimization)
8. **Pages Router (legacy) confuse karna** App Router se — yahan sirf App Router
9. **Heavy libraries Server Component mein import karna** — unnecessary bundle
10. **`Link` ki jagah `<a>` tag** — prefetching aur client navigation lose
</Callout>
